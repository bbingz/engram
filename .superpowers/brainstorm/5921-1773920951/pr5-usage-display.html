PR5: 用量显示方案对比

Claude 有 Opus(5h+周) / Sonnet(独立) / Spark(5h+周)，如何紧凑展示？

方案 A: 双行紧凑条

每个模型一组，上行 5h 下行 周，同色系深浅区分

Opus resets in 2h 15m · Spark resets in 45m

Pros

  • 所有信息一目了然
  • 深浅色区分 5h/周 直观
  • 5h 粗条+周 细条，主次分明

Cons

  • 占垂直空间较多 (4 模型约 120px)
  • Popover 空间紧张时可能拥挤

方案 B: 最紧急优先 + 展开详情

默认只显示每个模型最紧急的那个窗口，点击展开全部

  • 默认很紧凑 (4行)
  • 展开后信息完整
  • 只看最紧急，减少认知负担

Cons

  • 收起时丢失周用量信息
  • 多一次点击才能看全貌

方案 C: 分段条 — 单条双窗口

一个条里用两段表示：深色 = 5h，浅色 = 周

  • 一眼看到两个维度

  • "72/35%" 文字不够直觉

我的推荐: 方案 B

Popover 空间有限，默认展示最紧急的那个窗口，用户随时可展开看全部。 主窗口 Sources 页有充足空间，永远显示方案 A 的完整双行视图。

Popover = 方案 B (紧凑 + 可展开)
主窗口 Sources 页 = 方案 A (完整双行)
