PR7: 会话恢复工作流

从 GUI 和 CLI 两个入口恢复 AI 会话

入口 1: GUI — 主窗口 Resume 按钮

点击顶部 Resume 或右键会话 → Resume
Resume Session

●
Claude Code
claude --resume abc123
detected
Launch in:
Terminal iTerm2 Ghostty
Cancel Resume

入口 2: CLI — 命令行恢复

终端: engram --resume
$ cd ~/Code/coding-memory
$ engram --resume
Scanning sessions for /Users/bing/Code/coding-memory...
Recent sessions in this project:
1. Claude · 重构 Button 组件 · 2h ago · 23 msgs
2. Gemini · Fix login bug · 5h ago · 8 msgs
3. Cursor · Add API endpoint · 1d ago · 45 msgs
Select session (1-3) or q to quit: 1
Launching: claude --resume abc123def456...
Session resumed.

支持矩阵

工具 Resume 命令 检测方式
Claude Code claude --resume <id> 或 --continue which claude + 版本探测
Codex codex --resume <id> which codex + 版本探测
Gemini CLI gemini --resume <session-file> which gemini
Cursor 打开项目目录 (无直接 resume) which cursor / open -a Cursor
Others 可扩展: adapter 可选实现 resumeCommand() 按工具而定

实现架构

ResumeCoordinator (Node) 探测 CLI 版本 → 构建命令 → 返回 resume 指令。每个 adapter 可选实现
HTTP API POST /api/session/:id/resume → { command, args[], cwd, tool } 或 { error }
Swift TerminalLauncher 通过 AppleScript/NSWorkspace 启动 Terminal/iTerm2/Ghostty 执行命令
CLI 入口 (engram --resume) Node CLI 脚本。查询 daemon API 获取当前目录的会话列表，用户选择后直接 exec 恢复命令
Fallback CLI 不支持 resume → 打开项目目录 + 提示手动恢复。不支持的工具灰显
