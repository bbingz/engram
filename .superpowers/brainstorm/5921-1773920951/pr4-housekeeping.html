PR4: Agent 会话梳理 — 空会话与噪音分类

增强 session tier 系统，识别无意义会话并提供灵活过滤

当前 vs 目标

现在: Engram Tier 系统
4 个 tier，单维度分类:
skip — DB only，不显示
lite — +FTS，可搜索
normal — +embedding
premium — +Viking+summary
过滤: 单一 noiseFilter picker
(all / hide-skip / hide-noise)
问题: 不识别 preamble-only 会话
问题: 不识别 "无回复" 会话
问题: 不区分 probe 会话
问题: 过滤粒度太粗
目标: 多维度分类 + 细粒度过滤
保留 tier + 新增 3 个分类标签:
empty 0 条消息 (文件存在但无内容)
preamble 只有系统提示词 (CLAUDE.md 等)
no-reply 有用户输入但无 AI 回复
normal 正常对话会话
每个标签可独立过滤
默认隐藏 empty + preamble
用户可选择显示

分类逻辑 (Node 侧 session-tier.ts 扩展)

标签 判定条件 默认显示
empty messageCount == 0 或文件 < 1KB 隐藏
preamble 所有 user 消息匹配 preamble 模式 (CLAUDE.md, system-reminder, environment_context, agents.md 等) 且无 assistant 消息 隐藏
no-reply 有真实 user 消息，但 assistantMessageCount == 0 且 toolMessageCount == 0 显示 (可能是中断的会话)
normal 有 user + assistant 对话 显示

Preamble 检测模式 (参考 AgentSessions)

识别为 preamble 的内容特征
// 文件标记
"CLAUDE.md", "AGENTS.md", "GEMINI.md"
"environment_context", "system-reminder"
"<instructions>", "</instructions>"
// 系统角色声明
"you are an expert", "your role is"
"system:", "# System Instructions"
// 长 markdown 块 (>6行且>4个 heading/bullet)
通常是注入的项目说明或规则文档
