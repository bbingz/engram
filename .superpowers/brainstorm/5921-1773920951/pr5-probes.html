采集策略 (分级)

工具 采集方式 频率 需要配置
Claude OAuth API 或 tmux probe (fallback) 5 min OAuth token (自动从 ~/.claude/ 读取)
Codex tmux headless probe → /status 10 min tmux 已安装
Gemini Google API 配额端点 (如可用) 15 min API key (Settings)
Cursor 读取本地 usage/state 文件 5 min 无
Others 可扩展: 每个 adapter 可选实现 probeUsage() 自定义 按工具而定

实现层级

层 位置 职责
UsageProbe 接口 src/core/usage-probe.ts 定义 probe() → { source, metrics[] } 接口，adapter 可选实现
UsageCollector src/core/usage-collector.ts 定时器调度，调用各 probe，写入 DB + 发 stdout 事件
usage_snapshots 表 src/core/db.ts migrate() source, metric_name, value, unit, reset_at, collected_at
HTTP API src/web.ts GET /api/usage → 最新各工具用量快照
Daemon 事件 daemon stdout {"event":"usage", "data":[...]} 推送给 Swift
Swift UI PopoverView + SourcePulseView 用量条显示，告警 (>80% 变红)

关键设计决策

渐进式 先支持 Claude + Codex (有成熟参考)，其余工具后续 PR 扩展
Graceful probe 失败不影响主功能，静默跳过 + 日志记录
Probe 隔离 tmux probe 用专用目录，PR4 的增强 tier 会自动将其分类为 skip
用量告警 >80% 用量条变红，Popover 状态栏闪烁提示
