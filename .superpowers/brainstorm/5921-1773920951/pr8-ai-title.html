架构设计

标题生成流程
// 触发条件: 新会话索引完成 + 有 ≥2 条消息 + 无自定义标题
1. 提取会话前 3 轮对话 (user + assistant)
2. 构建 prompt:
"Generate a concise title (≤30 chars, Chinese or English
matching the conversation language) for this conversation."
3. 调用配置的 API (多 provider 支持)
4. 写入 sessions.summary 字段
5. Swift 侧 displayTitle 自动更新

实现要点

与现有 summary 复用 AI 标题写入 sessions.summary 字段。已有的 Viking summary 逻辑不冲突（summary 是长摘要，title 是短标题），可新增 generated_title 列
手动覆盖 用户的 customName 优先级最高: customName > generated_title > summary > 首条消息
批量回填 "Regenerate All Titles" 按钮触发后台 backfill，带速率限制 (1 req/s API, 并行 for Ollama)
Token 效率 只发前 3 轮对话，每轮截断 200 字。标题生成 prompt 约 500 tokens/次
