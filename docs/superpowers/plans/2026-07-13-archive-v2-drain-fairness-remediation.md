# Archive V2 Drain Fairness Remediation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore sustained HQ and M1 replication progress during a large local archive backlog by bounding the extra replica-wide transient pause and alternating which side of the shared ten-second drain pass receives first admission.

**Architecture:** Keep durable per-row retry deadlines, the single archive pipeline, and the existing event-driven drainer. `ArchiveReplicationCoordinator` will hold its process-local transient circuit breaker for 60 seconds independently of row jitter. `ArchiveV2ServiceCoordinator` will flip a process-local `remote`/`local` priority as soon as each pass acquires the pipeline and execute the existing stages in that order. The existing status DTO and settings card will expose only bounded scheduler state: effective per-replica pause metadata and the next pass priority.

**Tech Stack:** Swift 5.9, Swift concurrency actors/tasks, Foundation dates and Codable, GRDB/SQLite, XCTest, SwiftUI, String Catalog localization, Xcode/XcodeGen.

## Global Constraints

- Preserve immutable CAS objects, manifests, receipt formats, dual-receipt verification, row-level exponential full-jitter deadlines, stale-claim recovery, recovery drills, reclamation gates, and zero-delete behavior.
- Absent manual retry or resume, a transient infrastructure failure pauses unrelated rows for that replica for exactly 60 seconds even when row jitter is shorter; `remote_auth_rejected` and `replica_configuration_failure` remain indefinite attention pauses.
- HQ and M1 remain independent, with at most one active claim processor per replica and two globally.
- Preserve the single archive pipeline, existing ten-second pass deadline, two-second productive cool-down, cancellation checks, Low Power Mode and thermal gates, writer-gate admission, and the existing 32-locator/128-MiB capture budget.
- Alternate first admission by pass: process start is `remote`; then `local`; then `remote`. Flip once immediately after pipeline acquisition, even if a later stage throws, is cancelled, or consumes the deadline.
- Status pause reasons are exactly `transientInfrastructureBackoff` and `needsAttention`. Transient pause requires `pausedUntil`; attention pause forbids it. `nextPassPriority` is exactly `remote` or `local`.
- Older wire payloads decode with no replica pause and `nextPassPriority == "remote"`. Do not add polling, persistence, user tuning, dependencies, schema changes, or RemoteServer protocol changes.
- Keep all new English settings strings fully translated to Simplified Chinese.
- Do not modify `macos/Engram.xcodeproj` directly; run XcodeGen only if file membership changes.

unchanged while no second replica request starts before the 60-second breaker

Extend the existing manual retry/resume tests to prove clearing one replica removes both its transient and attention pause without clearing the other replica.

Add a pre-cancelled backlog pass after establishing a pause and assert the

- [ ] **Step 4: Implement one-flip alternating admission**

Initialize `nextBacklogPassPriority` to `.remote`. Immediately after `acquirePipeline(indexPriority: false)` succeeds, capture the current value and flip the stored value before adapter resolution or any fallible work. Extract the existing remote replication block into one actor-isolated helper so the pass can call it before or after the unchanged local capture/index/reconcile sequence without duplicating logic.

Remote-priority order is replication then capture/index/bind/policy. Local-priority order is the existing capture/index/bind/policy then replication. Both orders share the same deadline closure and the existing operation limits. Do not overlap local stages with replication.

- [ ] **Step 5: Add bounded pause state and validated wire fields**

Cache the latest effective pause state in the service coordinator whenever a replication attempt runs:

```swift
private struct ReplicaPauseState: Sendable {
    let reason: String
    let until: Date?
}
```

Map `retryPausedUntilByReplica` to `transientInfrastructureBackoff` plus an ISO-8601 deadline, and `pausedReplicaIDs` to `needsAttention` without a deadline. Attention takes precedence if both appear. Clear stale cache entries when a later replication result omits them; if no replication was admitted in a pass, retain the last known effective state. Clear the requested cache entry after an accepted manual retry.

Extend both Codable DTOs with strict initializer validation and `decodeIfPresent` defaults. Encoding uses the new fields; decoding older payloads stays valid. Pass the actor's current priority and pause state through `statusResponse` and `replicaStatus` without adding paths, identifiers, URLs, raw errors, or content.
