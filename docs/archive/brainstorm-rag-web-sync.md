# 头脑风暴：RAG + Web服务器 + Mac Mini同步

## 一、RAG 向量搜索实现方案

### 方案A: sqlite-vec (推荐)
- **优点**: 纯SQLite扩展，零额外依赖，与现有DB完美集成
- **缺点**: 需要编译加载扩展，大数量级时性能未知
- **嵌入源**: Ollama (本地) / OpenAI (在线)
- **索引策略**: 增量索引，后台异步生成

### 方案B: LanceDB
- **优点**: 专为AI设计，性能优秀，支持元数据过滤
- **缺点**: 单独数据库文件，需要学习新API
- **适用**: 大数据量场景

### 方案C: 纯内存向量 (不持久化)
- **优点**: 简单，启动时从会话生成
- **缺点**: 启动慢，内存占用大，不能持久化
- **适用**: 快速原型

### 方案D: 外部向量服务 (Pinecone/Milvus)
- **优点**: 专业托管，高性能
- **缺点**: 依赖外部服务，复杂度高
- **适用**: 不考虑，违背本地优先原则

### 嵌入模型选择
1. **Ollama + nomic-embed-text** (本地，免费，效果好)
2. **Ollama + mxbai-embed-large** (质量更高， slower)
3. **OpenAI text-embedding-3-small** (在线，便宜)
4. **混合策略**: 本地优先，本地不可用时fallback到在线

### 索引触发策略
- A) 后台定时扫描新会话
- B) 索引时即时生成 (慢)
- C) 用户点击"生成向量索引"按钮
- D) 混合：小会话即时，大会话后台

## 二、Web 服务器实现方案

### 框架选择
1. **Hono** (推荐) - 已在依赖，轻量，支持middleware
2. **Express** - 成熟，但较重
3. **原生 Node http** - 无依赖，代码多
4. **Fastify** - 性能最好，需新增依赖

### 功能范围

#### 最小可用 (MVP)
- GET /api/sessions - 列表
- GET /api/sessions/:id - 详情
- GET /api/search?q=xxx - 搜索
- GET / - 简单HTML界面

#### 完整功能
- 分页、排序、过滤
- 语义搜索接口
- 会话详情页 (markdown渲染)
- 暗色/亮色主题
- 移动端适配

### 安全方案
1. **无认证** - 仅内网，信任家庭网络
2. **Token认证** - URL参数或Header
3. **IP白名单** - 仅特定IP可访问
4. **只读模式** - 仅查看，无修改操作

### UI技术栈
1. **纯HTML+CSS+JS** - 零构建，简单
2. **HTMX** - 服务端渲染，交互好
3. **React/Vue** - 太重，需要构建
4. **原生+少量Alpine.js** - 轻量，有交互

## 三、Mac Mini 同步方案

### 触发方式
1. **LaunchAgent + cron** - 定时30分钟
2. **手动触发** - Settings里加按钮
3. **文件系统事件** - 检测到网络可达时自动同步
4. **应用启动时** - 启动时同步一次

### 同步策略
1. **单向拉取** (当前) - 仅从Mac Mini拉取
2. **双向同步** - 需要冲突解决策略
3. **主从复制** - 指定一台为主

### 网络检测
- 同步前ping 192.0.2.10
- 失败时记录日志，不重试

### 用户体验
- Settings显示上次同步时间
- 显示同步状态/进度
- 手动触发按钮
- 同步日志查看

## 五、风险与规避

| 风险 | 影响 | 规避方案 |
|------|------|----------|
| sqlite-vec编译失败 | RAG不可用 | 提供纯FTS5 fallback |
| Ollama未安装 | 本地嵌入失败 | OpenAI fallback + 提示安装 |
| Web端口冲突 | 服务启动失败 | 自动尝试其他端口 |
| 同步脚本失败 | 数据不一致 | 日志记录，手动重试 |
| 向量索引慢 | 用户体验差 | 后台异步，进度显示 |

---

## 六、决策建议

### 推荐组合:
1. **RAG**: sqlite-vec + Ollama (nomic-embed-text) + OpenAI fallback
2. **Web**: Hono + 纯HTML/JS + 无认证(内网)
3. **同步**: LaunchAgent 30分钟 + 手动按钮 + 启动同步

### 关键决策点:
- Web服务器端口: 3457 (与MCP 3456相邻)
- 向量维度: 768 (nomic-embed-text)
- 同步时机: 30分钟定时 + 启动时
- 索引策略: 后台异步，增量更新
